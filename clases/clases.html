Primera clase

¿Cómo funciona un sitio web? Podemos encontrar dos grandes áreas en el desarrollo,las que son el desarrollo backend y el desarrollo frontend. El desarrollo backend trabaja del lado del servidor mientras que el desarrollo frontend trabaja del lado del cliente en un navegador. En el desarrollo fronted se utiizan 3 lenguajes, los que son html, css y javaScript (sólo este último es un lenguaje de programación). HTML brinda la estructura y el contenido, css brinda los estilos y javaScript define el comportamiento (interacciones).

Tipos de sitios

Páginas estáticas: son los productos webs que tienen información para consumir como texto, imágenes, video, etc y que esta información no va a cambiar.Se puede agregar o quitar contenido pero en su mayoría será estático. lo mas importante de estos sitios es que no están conectados a un servidor o a una base de datos, es decir el usuario desde el navegador no podrá generar nigún tipo de cambio en el sitios.

Páginas dinámicas: Estas dejan de ser sitios y se convierten mas en aplicaciones web por todo lo que se puede hacer en ellas. Estas si están cpnectadas a una base de datos porque guardan información y los usuarios pueden interactuar en el sitio generando cambios.

¿Cómo es el proceso para construir un sitio web? Se utiliza la Metodología de diseño centrado en el usuario que conta de 5 etapas.

  • 1) Investigación:Se comienza con una investigación de usuarios, escenarios y compntencia(benchmark).
  • 2) contenidos: La información reunida se ordena, clasifica y jerarquiza en un mapa arquitectura de contenidos.
  • 3) wireframes: se crean los bosquejos del producto digital para ponerlo a prueba y medir la experiencia de usuario.
  • 4) html, css, js: Luego de haber puesto a prueba el wireframe este se contruye con los lenguajes del desarrollo frontend.
  • 1) Sitio web: Posteriormente el sitio es trabajado por el desarrollo backend, asegurando estándares de escalabilidad, estabilidad y seguridad.

Herramientas para el desarrollo frontend

  • 1) Editor de código: existen diversos editores de código como sublime text, visual studio code, atom, brackets, etc.
  • 2) Github: Controlador de versiones.
  • 3) devtools: Las herramientas de dearrollador que permiten inspeccionar el código mientras se visualiza en el navegador.

Segunda clase

¿Qué es html? Sus siglas HTML Hyper Text Markup Languaje significan lenguaje de marcado para hipertexto. No es un lenguaje de programación. Nos sirve para estructurar un sitio web.

¿Cómo se construye html? HTML se trabaja en un editor de texto como por ejemplo: visual studio code, sublime text, atom o incluso un block de notas. Pero herramientas como word o google docs no sirven para escribir html pues añaden código extra que no necesitamos. Para la creación de este sitio se utilizó visual studio code.
Si ya tenemos el editor de texto podemos pasar a escribir nuestra estructura html. Para poder escribir correctamente en este lenguaje utilizamos los símbolos mayor que < y menor que> al principio y al final de cada grupo de texto, generando elementos llamados etiquetas.


Hay que tener en consideración que no todas las etiquetas se cierran.

Comentario y anidación: Hacer un comentario en html es una manera de dejar mensajes a otra persona que lea nuestro código o inclusive recordatorios para nosotros mismos que el editor ignorará.La anidación hace referencia a que las etiquetas se pueden anidar unas fentro de otras.

Indentación: La indentación es una manera de ordenar los niveles de etiquetas que se generan con la anidación. La indentación permite una mejor lectura del código y hace parte de las buenas prácticas de codificación. Para indentar el código utilizamos la tecla tabulador y tambien existe un shortcut (atajo de teclado) que nos ordena el documento html completo correspondiente a la combinación de las teclas control+shift+p, aparecerá entonces en el editor la opción formato de documento, se presiona y el documento se ordena completamente.

Atributos: Los atributos son son palabras especiales utilizadas dentro de la etiqueta de apertura, para controlar el comportamiento del elemento.



Recomendaciones para nombres de archivos y carpetas

  • Prefiere minúsculas para nombrar archivos.
  • Utiliza sólo números, letras y guiones. Evita usar carácteres inusuales, tildes, símbolos, comillas, etc.
  • Utilizar guiones para separar las palabras y no los espacios.
  • Definir nombres cortos y relevantes.

Estructura de un documento html: Los sitios web pueden ser muy diferentes entre unos y otros, sin embargo todos deben respetar una estructura base del documento html. Se comienza con una etiqueta que se cierra a si misma y con la cual indicamos que tipo de documento escribiremos en el editor de texto, esa etiqueta se llama doctype. Luego de indicar el tipo de documento debemos colocar la etiqueta lang ,con ella le decimos al archivo que lenguaje utilizaremos, en nuestro caso español.

Posteriormente pasamos a definir las dos secciones mas grandes del documento, ellas son head y body. En Head se colocan los metadatos del sitio, es decir, aquellos datos que no son visibles para el espectador pero que dan indicaciones fundamentales para que funcione correctamente. En esta parte se vinculan las cascadas de estilos de css, los archivos javaScript, entre otras características que se pueden asociar. En el body en tanto se ubicará todo el contenido que si será visible por el espectador del sitio.



Tercera clase

Conociendo css. La abreviación css significa cascading style sheets, es un nombre descriptivo del comportamiento del lenguaje css, pues éste se comporta como una cascada leyendose de arriba hacia abajo pudiendo sobreescribirse en ocasiones.

¿Cómo se escribe css?El lenguaje css cuenta con tres partes importante que son:

  • Selector
  • Propiedad
  • Valor

Selectores css:Los selectores permiten llamar elementos que se encuentran en el html para añadirles estilos. Existen varios selectores con distintos grados de especificidad. La especificidad es el criterio que utiliza css para decidir cual instrucción se ejecuta por sobre otra.

Propiedades en css:Las propiedades en css son aquellas cualidades que se pueden declarar para un elemento seleccionado con un tipo de selector


Valor en css:La propiedad y el valor funcionan como llave cerradura, es decir la propiedad dice cual es la cualidad que recibirá estilos y el valor dice cuanto valen esos estilos.

Recomendaciones de escritura css

  • Indentar declaraciones dentro de la regla.
  • Utiliza sólo un selector por línea.
  • Dejar un espacio entre l último selector y la llave de inicio.
  • Dejar un espacio entre los dos puntos y el valor

¿Cómo vincular un archivo css a un arcivo html? Se utiliza la etiqueta link en la parte del head de html, allí se agrega el atributo hrf y en el se añade la ruta del archivo css.


Pseudoclases: permite seleccionar al elemento en un estado especial.La pseudoclase se añade a un selector y se inicia con un dos puntos.

Pseudoelemento: permite seleccionar una parte concreta un elemento. El pseudoelemento se añade a un selector y se inicia con dos dos puntos.


Herencia Indica si el valor de una propiedad es heredado de sus elementos padres.


Cuarta clase

Variables Las variables en CSS, son una manera de personalizar una propiedad, estos son definidas por el autor del CSS y contienen valores específicos que se pueden volver a utilizar en un documento, sin tener que volver a escribir la propiedad cada vez que se requiera, evitando posibles errores en el código.Se recomienda declarar en la raiz. del documentos css con la pseudoclase root y se utilizan con la función var().

¿Cómo cambio la tipografías de un sitio web? Para cambiar la tipografía de un sitio primero debe instalarse la nueva tipografía en nuestro equipo. La familia tipográfica se cambia en css con el atributo "font-family".Si la tipografía será utilizada en varias ocasiones dentro del sitio es recomendable agregarla a una variable.

Tamaños y espaciados del texto

  • Font size: Es la cualidad que define el tamaño del texto.
  • Line-height: Es la cualidad que define el interlineado, se declara con fracción.
  • Letter-spacing: Es la cualidad que define el espaciado entre letras.
  • word-spacing: Es la cualidad que define el espaciado entre palabras.
  • Font-weight: Es la cualidad que define el peso tipográfico.
  • Font-style: Es la cualidad que define el estilo de la fuente.

Alineación del texto Existen diferentes maneras de alinear un texto. Si se quiere alinear un texto se utiliza la propiedad text-align. El valor definirá la dirección a la cual se alineará el texto. Si se quiere alinear a la izquierda se utilizan los valores "left" o "start", si se quiere alinear a la derecha se utilizarán los valores "right" o "end". Para un texto centrado se utiliza el valor "center" y para un texto justificado se utiliza el valor "justify".>

Añadir color con css. Para utilizar la cualidad color existen muchos valores posibles. Se pueden utilizar con una palabra clave(variable), en formato hexadecimal,en rgb o rgba, en hsl o hsla y transparent/currentColor. Se recomienda que el color sea utilizado con variables.

Otras propiedades relacionadas

  • Text-decoration: Subrayado.
  • Text-shadow: Sombra.
  • Text-overflow: Desbordamiento
  • White-space: Uso de blanco dentro de un elemento.

Modelo de caja Este modelo explica que en css todo debe visualizarse como si fuera una caja. La página tiene dos formas de fluír, fluye como caja hacia abajo o fluye como línea. La propiedad display permite controlar como un elemento se incorpora dentro del flujo.

Opciones de display

  • Display-block: El elemento va a ocupar todo el ancho disponible y si no le damos un alto va a crecer lo necesario del contenido.
  • Display-inline:Usa el espacio justo según el contenido que tiene.
  • Display inline-block: el elemento tendrá un comportamiento combinado de los dos anteriores.

Quinta clase

Posiciones Las posiciones en cascada de estilo permite modificar desde donde se ubica un elemento. La posición predeterminada es "position:static" pero ésta puede variar con los valores "relative", "absolute", y "sticky". La declaración position:fixed sirve para ubicar elementos dentro de la ventana, aunque uno haga scroll el elemento se quedará en la posición indicada.

¿Qué es flexbox? Es un tipo de display que cnvierte un contenedor normal en un contenedor flexible. Entrega a un contenedor la habilidad para alterar el ancho, alto y espaciado de sus elementos para ajustarse lo mejor posible al espacio disponible. Un contenedor felxible expande sus elementos para rellena el espacio libre o los comportamiento para evitar que rebasen el área prevista.Los contenedores flexibles funcionan en un solo eje, vertical u horizontal. La dirección por defecto es horizontal y está definida por la propiedad "flex-direction" y por el valor "row", para cambiar a un eje vertical se debe colocar el valor column.

Vocabulario básico para hablar de flex

  • Contenedor flexible: Elemento con propiedad display:flex;
  • Elementos flexibles: Descenientes directos de un contenedor flexible.
  • Eje principal: Se declara con flex-direction, predertimanadamente es el eje x(sobre el cual se suceden los elementos flexibles).
  • Eje transversal: Es el eje perpendicular al eje principal.
  • Línea: Los elementos flexibles pueden quedar contenidos en una línea o utilizar varias según su ancho.

Si quiero distribuir los elementos en el eje principal utilizo la propiedad justify-content y si quiero distribuirlos en el eje transversal se hace con la propiedad align-items.

Múltiples líneas: Por defecto los elementos de un contenedor flex se ajustan al ancho disponible, uno al lado del otroen la misma línea. Esto puede ser modificado a través de la propiedad flex-wrap, permitiendo que estos rompan la línea y se ubiquen en una nueva línea según el ancho de los elementos.

Distribución Para distribuir elementos se utiliza la propiedad align. Se utiliza align-items para la alineación de elementos en el eje secundario (vertical) y se utiliza align-self para la alineación del propio elemento en el eje secundario, se aplica a un elemento flexible no a un contenedor.

Dimensiones Flexbox permite definir reglas de dimensiones para los elementos orientadas a daptar los tamaños para distintas resoluciones. Se recomienda que los contenedores no tengan un tamaño fijo. Mejor trabajar con propiedadescomo max-widht y max-height.

Sexta clase

CSS grid: Funciona desde el padre para darle diagramación a los elementos internos pero con mucho mas control y precisión de lo que se puede hacer. Una diferencia fundamental que tiene grid con flex es que permite hacerlo en dos dimensiones en lugar de una sola dimensión. En flex es siemprehacia el lado o hacia abajo. Al utilizar grid se piensa en como se divide la interfaz en áreas. Para grid no se suele utilizar tamaño en pixeles, en vez de pixeles se utiliza una medida que es propia de grid 1fr (fracción).

Complemento para flexbox: Mientras flexbox es un sistema de diagramación pensando en una dimensión, css grid permite pensar en un sistema de diagramación de 2 dimensiones. permite controlar tanto la posición como las dimensiones de los elementos en una grilla.

Soporte en navegadores: El trabajo en este módulo comenzó el 2012 a partir de una iniciativa de microsoft. Actualmente el soporte de navegadores es superior a un 97% globalmente.

Características de css grid

  • diagramación en dos dimensiones.
  • Permite definir una estructura de diagramación independiente del orden en que se encuentre el contenido.
  • En combinación con media-queries permite estructuras flexibles dependientes de las características del cliente (escritorio, móvil, tablet, etc).
  • Permite solapar áreas.

Vocabulario para css grid

  • Contenedor de grilla:elemento exterior, padre directo de los elementos de grilla.
  • Elementos de grilla: descendientes directos del contendor de grilla.
  • Línea: línea divisoria de la grilla.
  • Celda: área delimitada por dos flias y dos columnas adyacentes.
  • Track (pista):área delimitada por 2 líneas
  • Área: área delimitada por 4 líneas.
  • Unidad fr: representa una frácción del espacio restante en el contenedor de la grilla.
  • Espacio restante: espacio que no está asignado a un track o un área.

La respuesta es con media queries, estos son un tipo de código que se coloca en el css, el cual da la instrucción de adaptar los elementos según medidas definidas con media queries, es decir cada vz que el tamaño de dispositivo sea mas grande o mas pequeño que la medida definida el tamaño de los elentos cambiará.

Vocabulario asociado a javaScript

  • Variables: Manera de establecer un valor que después reutilizado en otros lugares.Hay distintos tipos de variables como strings, números, booleanos(una variable que vale verdadero o falso.). Los booleanos permiten hacer eventos condicionados.
  • Arreglos: Son objetos similares a una lista cuyo prototipo proporciona métodos para efectuar operaciones de recorrido y de mutación. Para escribir un arreglo se utilizan corchetes, todo lo que vaya dentro de los corchetes son objetos de un array y se separan por comas.
  • undefined: dato no definido, esto ocurre cuando se intenta realizar alguna acción con una variable y esta no se encuentra definida.

¿Que es el preprocesamiento? En palabras simples es el motor oculto que hace que lo ecribo de una forma pase a estar en la forma en que el navegador lo necesita.¿Que es un un preprocesador CSS?Es un programa que convierte un tipo de información en hojas de estilo CSS. Para css existen 3 librerías o framework que son less, sass y stylus que ofrecen una forma de escribir css.

Métodos de uso hay alguns aplicaciones de escritorio que permiten compilar sass o less como por ejemplo koala. En aplicaciones como esta, se les indica cual es la carpeta donde van a estar las hojas que se van a compilar y luego se indica cual va a ser el archivo final.

Una segunda manera de compilar los archivos que no sea utilizar una aplicación de escritorio es utilizar una extensión de visual estudio. llamada live sass compiler que da la instrucción a un archivo sass para que se compile en un archivo x, de esta manera se logra hacer todo dentro de visual estudio.

SCSS Ventajas La anidación de declaraciones: Es posible anidar los selectores de los elementos descendentes, lo que permite visualizar mejor la jerarquía de los elementos.Debe utilizarse cuidadosamente, ya que la especificidad de los elementos puede aumentar si darnos cuenta.

SCSS Variables Tal como las variables CSS, cada preprocesador cuenta con la alternativa de crear variables reutilizables. Las variables se escriben con el signo peso ejemplo $variable: #ccc; y deben estar declaradas (escritas) antes de ser utilizadas. Una ventaja que tienen en comparación con las variables de css es que hay ciertas funcionalidades que no se pueden hacer con css por ejemplo trabajar colores.

Operaciones Otra cualidad que consideran casi todos los preprocesadores es poder realizar operaciones matemáticas dentro del valor de una propiedad. SCSS considera al menos +, -, *, /, y %

Funciones Sass ofrece una larga lista de funciones que podemos utilizar para gestionar colores, hacer cálculos o programar funciones más complejas.

scss Import Una de las principales ventajas de un preprocesador es la capacidad de importar otros archivos scss (o css), lo que permite aislar estilos en hojas separadas.

scss Mixins Forma de crear funciones propias, para agrupar estilos comunes que puedan generarse a partir de variables. Podemos definir los parámetros y comportamientos internos a partir de ellos.

Discapacidades:

Situaciones:

¿Porqué es importante la accesibilidad?

  • Acceso a la información como derecho humano
  • Igualdad de acceso e igualdad de oportunidades para personas con distintas capacidades
  • Inclusión social: adultos mayores, población en áreas rurales, en países en vías de desarrollo...
  • Puede ser un requerimiento legal en ciertos contextos
  • Razones de negocio: abarcar un mayor mercado, cumplimiento de responsabilidad pública y políticas de inclusión

Principios de Accesibilidad Web

PerceptibleLa información y los componentes de la interfaz de usuario deben ser presentados a los usuarios de modo que ellos puedan percibirlos.

¿Cómo hacerlos perceptibles?

  • Alternativas de texto para contenido no textual.
  • Subtítulos y otras alternativas para multimedia.
  • Contenido que puede ser presentado de diferentes maneras.
  • Puede ser un requerimiento legal en ciertos contextos.
  • Contenido más fácil de ver y escuchar.

Operables Los componentes de la interfaz de usuario y la navegación deben ser operables.

¿Cómo hacerlos operables?

  • Funcionalidad disponible desde el teclado.
  • Los usuarios tienen tiempo suficiente para leer y utilizar el contenido.
  • El contenido no causa convulsiones ni reacciones físicas.
  • Los usuarios pueden navegar fácilmente, encontrar contenido, y determinar dónde están.
  • Los usuarios pueden utilizar diferentes modalidades de entrada además del teclado.

Comprensible La información y el manejo de la interfaz de usuario deben ser comprensibles.

¿Cómo hacerlos comprensibles?

  • El texto es legible y comprensible.
  • El contenido aparece y opera de forma predecible.
  • Asistencia a los usuarios para prevenir y corregir errores.

Robusto El contenido debe ser suficientemente robusto como para ser interpretado de forma fiable por una amplia variedad de aplicaciones de usuario, incluyendo las ayudas técnicas. El contenido es compatible con las herramientas de usuario actuales y futuras.

Niveles de conformidad

  • Nivel “A”:Alto impacto en una amplia variedad de grupos de usuarios.No está enfocado en un único tipo de discapacidad. Bajo impacto en la lógica de presentación o de negocios. Generalmente de fácil implementación..
  • Nivel “AA”: Alto impacto para los usuarios, pero pueden estar referidos a grupos o discapacidades específicas. Para cumplir con estos criterios podrían ser necesarios cambios a la presentación o lógica de negocio del sitio.
  • Nivel “AAA”: Enfocados en mejoras para poblaciones de usuarios específicas. Pueden ser más difíciles o costosas de implementar de acuerdo a las limitaciones de la plataforma. La proporción de costo/beneficio puede ser más baja.

Técnicas básicas para la accesibilidad En el código se debe utilizar el atributo alt en las imágenes. Se Debe utilizar marcado estándar y válido. Se debe validar datos de formularios e indicar forma de solucionar errores.
